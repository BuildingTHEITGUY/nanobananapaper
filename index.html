PaperBanana: Automating Academic Illustration

Mohamed Asath (Project Lead)
University of Dubai • IT Infrastructure & DevOps Initiative
Code Pilot

Executive Summary

Generating publication-ready illustrations is a critical bottleneck in the University of Dubai's research workflow. We are introducing the PaperBanana Agentic Framework to our institutional infrastructure. Unlike standard AI tools, this system orchestrates specialized agents to retrieve references, plan content, and write executable Python code for statistical plots.

Automating Complex Methodologies

Figure 1: Examples of "NeurIPS-Standard" diagrams generated automatically from raw text. This capability allows faculty to focus on science, not graphic design.

The Multi-Agent Architecture

PaperBanana does not rely on a single prompt. It uses a Linear Planning Phase followed by an Iterative Refinement Loop.

Figure 2: The 5-Agent System (Retriever, Planner, Stylist, Visualizer, Critic) ensuring quality control.

Data Fidelity: The "Anti-Hallucination" Engine

The Challenge

Standard Generative AI (like DALL-E) treats charts as "pictures," often inventing numbers ("Hallucination").

The Solution

PaperBanana's Coder Agent writes executable Python (Matplotlib) code. This ensures the plot is a mathematically perfect representation of the researcher's raw data.

Figure 3: Comparison showing how Code-Based Generation (Right) eliminates the data errors found in Image-Based Generation (Left).
